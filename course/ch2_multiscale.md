# Ch2: 多尺度异常定位 — 从全局分数到像素级热力图

## ❓ 灵魂拷问

Ch1 的 DINOv2 全局余弦距离告诉你"这张图可能有问题"。但比赛评分公式里，**分割得分占 50%**——你不仅要判断有没有缺陷，还要定位缺陷在哪个像素。

想象产线上的质检员：你的模型说"这个零件异常分数 0.7"。质检员看一眼零件——光洁如新。"哪里异常？" 你答不上来。

**不知道异常在哪里的检测 = 没有用的检测。** 本章从全局判断升级到逐像素定位。

## 📚 前置认知

### Ch1 回顾：你只有一个全局数字

Ch1 结束时，你有 50 个基线分数，工具是 `encoder(img) → [768维向量] → 余弦距离 → 1个数字`。

就像你打了个电话给仓库问"这批货有问题吗"，系统回答"可能性 13.3%"。你不知道是因为箱子破了还是标签贴歪了——只有一个模棱两可的数字。

### 从"话费余额"到"通话详单"

DINOv2 内部其实比你用的多得多。Ch1 只用它的"最终输出"——就像你只看话费余额，但运营商手里有完整的通话详单（每一通电话的时间、时长、对方号码）。

**DINOv2 内部是 Vision Transformer（ViT），工作原理如下：**

```
一张 448×448 的图片
  → 切成 32×32 个网格，每个小格 14×14 像素（共 1024 个小格）
  → 提取其中中心的 14×14=196 个小格作为 "patch"
  → 每个 patch 编码成 768 维的 "token"（类似一个词）
  → 加上 1 个 CLS token（全局摘要）+ 4 个 register token（辅助）
  → 总共 201 个 token，每个 768 维
  → 经过 12 层 Transformer 处理
  → 每层的 196 个 patch token 可以还原成 14×14 的空间特征图
```

> 💡 **类比：** 你有一张班级合照。Ch1 只看照片背面写的"三年二班"（CLS token）。本章翻到照片正面，逐个看每个学生的脸（patch token）。

### 为什么用"多层"而不是一层

就像人眼看东西：
- **第一眼（浅层 Layer 3）：** 看到颜色、边缘、纹理——"这个是红色的，那个是金属的"
- **第二眼（中层 Layer 6）：** 看到形状、部件——"这里有个螺丝孔，那边是散热片"
- **第三眼（深层 Layer 9, 11）：** 看到整体语义——"这是一块完整的 PCB 板"

单层只能看到一个层次的信息。融合多层 = 既有细节又有全局，异常检测更准确。

### 本章与 Ch4（完整 INP-Former）的区别

本章用手工规则（余弦距离）做 patch 匹配。Ch4 用**可学习**的 INP 原型和交叉注意力自动学习"什么算正常"。本章是理解 Ch4 的必要铺垫。

## 📖 主体内容

### 2.1 从 [768] 到 [768, 14, 14]：空间特征图怎么来的

Ch1 代码只用了 CLS token：

```python
feat = encoder(img)  # [B, 768] — CLS token，全局摘要
```

但 DINOv2 内部每个 Transformer 层都有完整的 patch token。要拿到它们，需要手动遍历层：

```python
# 不是简单的 encoder(img)，而是逐步走每一层
x = encoder.prepare_tokens(img)  # 初始化 token（201个）
for i, blk in enumerate(encoder.blocks):
    x = blk(x)  # 经过一层 Transformer
    if i in [3, 6, 9, 11]:  # 在第 3/6/9/11 层停下来
        patch_feats = x[:, 5:, :]  # 去掉前 5 个 token（CLS+4个register）
        patch_feats = patch_feats.reshape(B, 768, 14, 14)  # 还原为空间图
        features.append(patch_feats)
```

> 💡 **关键变化：** `[B, 768]` 变成了 `[B, 768, 14, 14]`。多了两个维度——14×14 的空间网格。每个位置 (i,j) 有自己独立的 768 维向量。你可以对左上角和右下角分别算异常分数了。

### 2.2 逐 patch 余弦距离：从特征图到热力图

有了 14×14 的空间特征图，异常检测变成3步：

```
步骤 1：对每个类别的训练集(全部正常)，逐层、逐位置计算特征均值
       → proto[layer] = [1, 768, 14, 14]
       → proto[3][0][3,5]  = "Layer 3 在位置(3,5)的"正常"特征长什么样"

步骤 2：对测试图片，同样提取 4 层特征图

步骤 3：逐层、逐位置计算余弦距离
       → anomaly[layer][i,j] = 1 - cos_sim(feat[layer][:,i,j], proto[layer][:,i,j])
       → 4 层平均 → 上采样到 448×448 → 最终热力图
```

```python
# 核心代码
for layer in layers:
    feat = features[layer]      # [1, 768, 14, 14]
    proto = prototypes[layer]   # [1, 768, 14, 14]
    
    # 把所有 196 个位置展平，逐一算余弦距离
    cos_sim = F.cosine_similarity(
        feat.permute(0,2,3,1).reshape(-1, 768),  # [196, 768]
        proto.permute(0,2,3,1).reshape(-1, 768)   # [196, 768]
    )
    anomaly = 1 - cos_sim  # [196] — 每个位置的异常分数
    anomaly = anomaly.reshape(1, 1, 14, 14)  # 还原空间结构
    anomaly = F.interpolate(anomaly, size=448)  # 上采样到原图大小
```

> 💡 **为什么 14×14 不是 448×448？** ViT 里一个 patch 对应原始图像 32×32 像素的区域。14×14 的异常图包含 196 个"小块"的分数。上采样到 448×448 只是平滑放大——热力图的物理分辨率是 32×32 像素，不是 1×1 像素。Ch4 的 INP-Former 解码器才真正把分辨率提到像素级。

### 2.3 读懂热力图：三种典型模式

以 `3_adapter` 为例，热力图呈现三种模式（见文末 **图2.1**）：

**模式一：高纹理区域天然"热"（假阳性陷阱）**

`3_adapter` 视角 0 的顶部有细密插针阵列。即使产品完全正常，插针之间的微小间隙和反射变化也让该区域的特征偏离均值。热力图上插针区域永远是"红"的——这不是缺陷，是高频纹理的固有特征。

**模式二：边缘和阴影自带异常**

任何物体的边缘、暗角、反光区域在特征空间中都会偏离中心。`DVD_switch` 的金属壳体反光区域，热力图一片红。但要问：如果所有正常样本都有这个反光，它算异常吗？

**模式三：均匀区域最"冷"**

纯色塑料面板、PCB 基板等大面积均匀区域，patch 特征与训练集高度一致，热力图接近全蓝——这是你想要的：模型说"这里完全正常"。

> 🔑 **核心教训：热力图上的红色不等于缺陷。** 它只告诉你"这个区域的特征和正常均值有差异"。这个差异是因为产品纹理（正常）还是缺陷（异常），当前方法完全分不清。这恰好是 Ch4 INP-Former 要解决的问题。

### 2.4 3类测试结果：看数字更要看图

| 类别 | 像素异常均值 | Ch1 全局分 | 倍数 |
|------|------------|-----------|------|
| `3_adapter` | 0.604 | 0.081 | 7.5× |
| `DVD_switch` | 0.667 | 0.184 | 3.6× |
| `D_sub_connector` | 0.623 | 0.147 | 4.2× |

> 🔑 **分数翻倍不是模型变差了——是度量变了。** Ch1 把整个热力图压成 1 个数，自然低。像素级保留空间分布，分数自然高。两套分数不可直接比较，它们衡量的是同一个事物的不同方面。

### 2.5 Ch1 → Ch2 → Ch4：三阶段能力对比

| 维度 | Ch1 全局 | Ch2 多尺度 | INP-Former (Ch4) |
|------|---------|-----------|-----------------|
| 输入 | CLS token | 4层 patch tokens | 8层 + INP原型 |
| 输出 | 1个分数 | 448×448 热力图 | 448×448 热力图 |
| 空间信息 | ❌ | ✅ | ✅ |
| 可学习参数 | 0 | 0 | ~3000万 |
| 纹理/缺陷区分 | ❌ | ❌ | ✅ |
| 分辨率 | 整图 | 32×32→上采样 | 像素级（经解码器） |

## 🔬 本章检查清单

- DINOv2 把图片切成多少个 patch？→ **196 个**（14×14），每个 patch 32×32 像素
- 多尺度用了哪几层？→ Layer 3（纹理）、6（形状）、9、11（语义）
- 热力图原始分辨率？→ 14×14，上采样到 448×448
- 热力图"红"的区域一定是缺陷吗？→ **不一定**，高纹理、边缘、反光都会红——这是当前方法最大局限
- 像素级分数为什么比 Ch1 全局分离得远？→ 度量不同：全局分压缩了空间分布，像素级保留了
- Ch4 INP-Former 解决了什么？→ 可学习原型能自适应区分"正常纹理"和"真正缺陷"

## ❓ 新问题

你的热力图在插针阵列上一片红——但你知道那里没有缺陷。余弦距离说"这里和原型不一样"，但它分不清这个"不一样"是因为产品固有特征还是真正的瑕疵。

这个问题的根源在 DINOv2 本身——它毕竟是在自然图片（狗、猫、风景）上训练的。为什么它对"没见过"的工业零件也能提取有意义的特征？里面的 register token 是什么？为什么不用原始版 DINOv2 而要用 `_reg` 版本？
