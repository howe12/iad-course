## 📖 主体内容

### 3.1 自监督学习：你不告诉我答案，我自己猜

监督学习大家都会：给猫的图片 + 标签"猫"，给狗的图片 + 标签"狗" → 训练。但 DINOv2 在 1.42 亿张图片上训练——谁去标注 1.42 亿张图？

**答案：不需要标注。DINOv2 用了一个聪明的"教师-学生"框架。**

```
同一张图片，做两种不同的随机裁剪：
  → Crop A（全局视角，占图片 50%+ 面积）
  → Crop B（局部视角，占图片不到 50% 面积）

教师网络看 Crop A → 输出特征向量 teacher_A
学生网络看 Crop B → 输出特征向量 student_B

训练目标：让 student_B 尽量接近 teacher_A
```

> 💡 **类比：** 你给一个小孩看一张照片的"半张脸"（局部），然后问他"这张脸属于哪张全景照？"。小孩通过不断对比半张脸和全景照，学会了脸的底层结构——眼睛、鼻子、嘴的相对位置。**他没学过"这是张三的脸"，但他学会了"脸的基本构成规律"。** DINOv2 同理——没学过"这是 3_adapter"，但学会了"零件的纹理、边缘、几何结构规律"。

**教师网络不是固定的——它在"自我进化"：**

教师网络的参数不是独立训练的，而是学生网络参数的**指数移动平均（EMA）**：

```python
teacher_params = 0.996 × teacher_params + 0.004 × student_params
# 教师 = 过去所有学生版本的"加权平均"
# 像一个慢慢积累经验的老师——不会被某一个学生的错误带偏
```

这个设计的精妙之处：教师网络天然比学生网络更稳定（EMA 抑制了噪声更新），而学生网络不断追赶更稳定的教师 → 两者一起提升，且整个过程**不需要任何标签**。

### 3.2 特征空间分布：用实际数据打开

说了一堆理论，回到我们的 5 个实际类别，看看 DINOv2 到底学到了什么。

我们从训练集中取 `3_adapter`、`battery`、`DVD_switch`、`circuit_breaker`、`effect_transistor` 各 3 个样本，送入 DINOv2 提取 768 维特征，用 PCA 降维到 2D 可视化（见文末 **图3.1**）。

**实验一：跨类别的特征相似度**

用每个类别的**平均特征向量**计算两两类别的余弦相似度：

| | 3_adapter | battery | DVD_switch | circuit_breaker | effect_transistor |
|---|-----------|---------|------------|-----------------|-------------------|
| **3_adapter** | 1.00 | 0.09 | 0.09 | 0.21 | 0.09 |
| **battery** | | 1.00 | 0.13 | 0.33 | 0.23 |
| **DVD_switch** | | | 1.00 | 0.28 | 0.27 |
| **circuit_breaker** | | | | 1.00 | 0.35 |
| **effect_transistor** | | | | | 1.00 |

> 🔑 **跨类平均相似度仅 0.21，类内 1.00，差距 0.79。** 不同类别的零件在 DINOv2 眼中是完全不同的东西——特征分离得非常好。这解释了为什么 Ch1 的余弦距离基线有效：正常原型只对"自己的类别"成立，换一个类别完全不对。

**实验二：同类样本的一致性（这直接解释 Ch1 的异常分数！）**

真正有价值的是同类内部的样本相似度——它直接决定该类的异常基线是高是低：

| 类别 | 同类样本平均相似度 | 标准差 | 最低相似度 |
|------|------------------|--------|-----------|
| `effect_transistor` | **0.987** | 0.002 | 0.985 |
| `circuit_breaker` | 0.977 | 0.004 | 0.973 |
| `DVD_switch` | 0.937 | 0.024 | 0.906 |
| `battery` | 0.827 | 0.028 | 0.796 |
| `3_adapter` | **0.371** | **0.418** | **0.075** |

> 🔑 **现在回头看 Ch1 的基线分数：** `effect_transistor` 的异常分数是 0.044（最低），`3_adapter` 是 0.081（中等偏高）。答案就在这里：`effect_transistor` 的样本特征紧得像一个点（相似度 0.99），任何偏差都极其微小 → 余弦距离小。`3_adapter` 的样本特征散得像一团雾（相似度 0.37，最低只有 0.075）——有的样本在特征空间里孤零零地飘在边缘 → 这些样本的余弦距离自然大。

**这不是模型的问题，是产品本身的特征。** DINOv2 忠实地反映了现实：封装芯片就是长得一模一样，塑料壳体就是每一批都有微妙差异。

### 3.3 特征随层深化的演变：从纹理到语义

DINOv2 有 12 层 Transformer。不同的层关注不同的信息。我们取同一张图的第 0/3/6/9/11 层的 patch token，看每层特征的活跃程度（见文末 **图3.2**）：

- **Layer 0（输入层）：** 基本就是原始像素值，没有结构
- **Layer 3（浅层）：** 开始出现边缘、纹理的轮廓。高频区域（纹理密集处）激活高
- **Layer 6（中层）：** 几何结构浮现——圆形、方形、孔洞的边界清晰
- **Layer 9（深层）：** 语义信息占主导——整体形状、部件归类
- **Layer 11（输出层）：** 特征高度抽象，不同区域的激活趋于均匀

> 💡 **这对异常检测的意义：** 浅层特征适合检测纹理缺陷（划痕、毛刺——局部纹理变了）；深层特征适合检测结构缺陷（缺件、形变——整体语义变了）。融合多层（Ch2 的做法）= 兼顾两者。

### 3.4 Register Token：为什么必须用 `_reg` 版本

你可能注意到了，所有代码用的都是 `dinov2reg_vit_base_14`，不是 `dinov2_vit_base_14`。`_reg` 是什么？

**问题背景：ViT 的"特征崩溃"现象**

原始 DINOv2 在某些图片上会出现一个诡异的问题：所有 patch token 变得高度相似（"坍缩"到少数几个方向），丢失了大量空间信息。这尤其发生在"干净"的图片上——大面积纯色背景、均匀纹理、无复杂遮挡的图片。

工业零件图片恰好就是这种"干净"图片——单一产品、纯色背景、光照均匀。

**解决方案：Register Token（Darcet et al., 2024）**

在输入的 token 序列中额外加入 4 个 **register token**——它们是"垃圾桶 token"，专门用来吸收那些"不知道该表达什么"的信息。

```
标准 ViT token:    [CLS, patch_1, patch_2, ..., patch_196]   (197 个)
Register ViT token: [CLS, reg_1, reg_2, reg_3, reg_4, patch_1, ..., patch_196]  (201 个)
```

这 4 个 register token 不参加分类，不参加下游任务，只在训练时"吸走"那些会让 patch token 坍缩的信息。训练完成后，patch token 更丰富、更 diverse。

> 💡 **类比：** 一个班有 30 个学生（patch token），老师要让他们各自发展特长。但有 4 个"万能学生"（register token）——他们不考试、不排名，专门负责做一些重复性杂务（吸收坍缩信息），让其他 30 个学生可以专心发展自己的方向。

**为什么工业场景必须用 `_reg`：** 没有 register token 的 DINOv2，在纯色背景的工业零件图上，可能有 50% 的 patch token 趋同。这意味着 Ch2 的热力图在这些区域完全无效——所有位置分数都一样，无法定位异常。
