# Ch1: 数据长什么样？— 加载、可视化与最简基线

## ❓ 灵魂拷问

你拿到了50个类别的训练集，每个类别20个样本，每个样本5个视角。看起来数据量不小——但等一下：**所有训练图片都是正常的。** 没有一张有缺陷。

这意味着你没法像传统分类任务那样 `model.fit(X, y)`——你没有 `y`。训练阶段能用的唯一信息是"这些样本都是正常的"。你的模型只能学会"正常长什么样"，然后在推理时问"这个样本和正常差多少"。

这不只是数据问题，这是一个**范式问题**。本章带你从零开始面对这堆只有正常样本的数据。

### 你加载的预训练模型在干什么

往后 8 章你反复用到两个东西，先搞清它们的分工：

**DINOv2 — 通用的「眼睛」（你不动它）**

- 在 1.42 亿张自然图片上训练过，冻结，只做前向推理
- 输入一张零件照片 → 输出 768 维特征向量 + 内部多层特征图
- 它不认识"电容"还是"电阻"，但它知道"纹理、边缘、形状"这些底层视觉概念
- 类比：一个没学过中文的外国人，能一眼看出招牌歪了——因为"歪"是跨语言的视觉概念

**INP-Former — 给眼睛装上「这个零件应该长什么样」的记忆（你加载官方权重）**

- 在 DINOv2 之上加了 6 个可学习的 INP token（正常原型）和 8 层解码器
- 训练时它看了 50 类 × 20 张正常照片，学会了每类零件的"正常纹理范围"
- 推理时，新图片的特征跟 6 个原型比较——差距大的像素就是异常
- 类比：DINOv2 只能说"这个区域不太对"，INP-Former 能说"这个连接器的 3 号针脚比正常原型偏了 0.3mm"

**为什么只做推理不做训练？** 官方已经把 INP-Former 在 Real-IAD + MVTec + VisA 上训练了 200 个 epoch，权重直接可用。你的任务是理解和运用它，不是重训它。

### 分数怎么变成排名

比赛看一个三合一分数：

```
S_total = 100 × (0.3 × S_cls  +  0.5 × S_seg  +  0.2 × S_zs)
                  ↑图像级分类      ↑像素级分割        ↑零样本泛化
               "图有没有缺陷？"  "缺陷在哪个像素？"  "未见过的类怎么判？"
```

**S_seg 占 50%——比赛最看重"定位缺陷位置"的能力，不只判断有还是没有。**

这就是为什么 Ch2 讲热力图、Ch4 讲 INP-Former 像素级推理、Ch5 讲分割指标：一切都是为了这 50% 的分数。

### 🔧 两个前置概念：搞懂"模型"和"向量"在说什么

在深入代码之前，先建立两个核心认知。

**概念一：图片 → 数字向量，就像快递单号**

DINOv2 做的事很简单：输入一张图片，输出一串768个数字。这串数字就是图片的"特征向量"。

> 💡 **类比：** 你把一张快递单拍下来发给仓库系统。系统不认识你的字迹，但它能把这张快递单压缩成一个"快递单号"——20位数字。两张内容相似的快递单（同一个发件人、相邻地址），单号接近。两张完全不同的单子，单号差距大。DINOv2 就是这个"生成单号"的系统——它不在乎你拍得清不清楚、光线好不好，它只关心图片的**内容特征**。

768这个数字不是拍脑袋定的。它是 DINOv2 的 `vit_base` 版本的特征维度。数字越大，能描述的细节越多（但也越占内存）。

**概念二：全局向量 vs 空间特征（本章 vs 下一章的关键区别）**

DINOv2 生成的 768 维向量是对**整张图**的一个全局描述。就像你说"这个零件整体看起来OK"——你知道它OK，但不知道具体哪一部分让你觉得OK。

下一章 Ch2 会用 DINOv2 **内部的**空间特征——它实际上会告诉你"左上角看起来OK，右下角有点怪"。这个区别贯穿全书。

> 📌 **记住这条线：** Ch1 用全局向量（整图判断）→ Ch2 用空间特征（逐位置判断）→ Ch4 用 INP-Former（自适应学习什么算正常）。

### 🔧 关键模型速览

本章和后续章节会反复提到两个模型。先搞清楚它们是什么：

**DINOv2 — 通用视觉编码器（Meta, 2023）**

- 全称：**D**iscriminative **I**mage **N**etwork with **O**ptimization v2
- 本质：一个**预训练好的图像特征提取器**，类似于 NLP 中的 BERT/GPT 但在视觉领域
- 训练方式：自监督学习，在 1.42 亿张图片上训练，**不需要人工标注**
- 输入：一张图片 → 输出：一个 768 维特征向量（`vit_base` 版本）
- 为什么用它：它在没见过的工业零件图片上也能提取有意义的特征——纹理、边缘、形状

> 💡 **类比：** 把一个不认识中文的人扔到中国生活了十年，他虽然不会读写，但能一眼看出街上的招牌"长得很正常"还是"有点不对劲"。DINOv2 就是那个在中国泡了十年的外国人——它从未见过这些零件，但知道正常视觉模式长什么样。

**INP-Former — 工业异常检测模型（本文核心方法）**

- 全称：**I**mage-**N**eutral **P**rototype **Former**
- 本质：在 DINOv2 基础上**加了可学习的"异常检测头"**
- 三层结构：`DINOv2 编码器` → `INP 原型匹配` → `解码器生成热力图`
- Ch1 只用 DINOv2（全局判断），Ch4 才用完整 INP-Former（像素级定位）

### 1.2 五视角对比：换个角度，同一个产品像换了张脸

以下是随机抽取6个类别的5视角对比图。同一行的5张图来自**同一个样本的5个不同角度**：

**（见文末 **图1.2**：6类5视角对比图）**

> 📷 **看图要点：** 从左到右是视角 0~4。同一行看水平差异（同产品不同角度），不同行看垂直差异（不同产品之间）。

为什么这张图重要？三个规律：

1. **V0（正上方）和 V4（斜上方）的光照完全不同** — 光线方向一变，同一条纹理的"亮度"就变了
2. **金属零件在 V1 角度反光最严重，V2 角度最暗** — 反光不是缺陷，是物理规律
3. **塑料件在不同角度下注塑纹理方向不同** — 纹理方向是制造工艺决定的，不是缺陷

> 💡 **核心教训：** 如果你单独看视角 1 就下结论，可能把正常的光影变化当成缺陷。**多视角信息是双刃剑——多了信息，也多了混淆。**

### 1.3 图片尺寸：为什么需要 Crop + Resize

50个类别的原始图片尺寸分布如下。横跨 398px ~ 4343px，11倍差异：

**（见文末 **图1.3**：50类图片尺寸分布图）**

> 📷 **看图要点：** 每个柱子是一个类别的图片尺寸（取视角0）。红线是 448px——模型的标准输入尺寸。柱子有的不到红线一半（`effect_transistor` 398px），有的是红线的近10倍（`circuit_breaker` 4343px）。

**极端例子：**
- 最小：`effect_transistor` (398×398) — 封装好的小芯片，拍满整张图。直接放大到 448px 没问题，细节不会丢。
- 最大：`circuit_breaker` (4,343×4,343) — 大体积断路器，相机贴得很近。直接缩到 448px 会把一根头发丝粗细的划痕压成 1 像素。

预处理流水线正是为解决这个问题：

```
原图 (398~4343px)
  → Resize(448, 448)   — 第一步缩放到统一尺寸
  → CenterCrop(392)    — 裁掉最外围的 28 像素（通常是背景/边缘）
  → Resize(448, 448)   — 恢复到标准输入尺寸
  → Normalize(ImageNet 均值/标准差) — 数值归一化
```

> ⚠️ **为什么不是直接 Resize 到 448？** 一张 4343px 的图片直接缩到 448px，缩了 9.7 倍。原本一排 10 像素的划痕变成 1 像素——完全不可见。先 Crop 392 再 Resize 448，意味着模型**只关注中心区域**（避免边缘背景干扰），同时在保留的分辨率上做了轻量缩放。

### 1.4 最简基线：DINOv2 + 余弦距离

不带 INP-Former，不训练。只用一个预训练的 DINOv2 Base 编码器。

**算法流程（3步，无训练，6分钟跑完 50 类）：**

```
第1步：对每个类别，把20个训练样本的100张图送入 DINOv2
       → 得到 100 个 768 维向量
       → 取均值 = 该类别的"正常原型" proto（读作"普若投"）

第2步：对每个测试样本的5张图，同样送入 DINOv2
       → 得到 5 个 768 维向量

第3步：计算每个测试向量与正常原型的余弦距离
       anomaly_score = 1 - cos_similarity(test, proto)
       → 距离越大 = 越可疑（越不像训练集里的"正常样子"）
```

> 💡 **类比：** 你开了一家餐厅，每天做同一道菜——麻婆豆腐。你把过去 100 盘"正常"麻婆豆腐的照片全部存档。今天炒了一盘，拍照和存档比——颜色偏红？可能是豆瓣酱放多了。花椒粉洒得不均匀？可能是手抖。每一点偏差都是一个"异常分数"的分量。**正常人都会说这是一盘麻婆豆腐，但老主顾能看出今天的和平时不太一样。**

```python
# 核心代码
encoder = vit_encoder.load("dinov2reg_vit_base_14").to("cuda")

# 提取所有正常样本的特征
with torch.no_grad():
    for imgs in train_dataloader:
        feats = encoder(imgs)  # [B, 768] — 每张图变成一个768维向量
        train_features.append(feats)

# 正常原型 = 所有正常样本特征的均值
proto = torch.cat(train_features).mean(dim=0)

# 对测试样本计算异常分数
for imgs in test_dataloader:
    feats = encoder(imgs)
    cos_sim = F.cosine_similarity(feats, proto.expand(B, -1), dim=1)
    anomaly_score = 1 - cos_sim  # 越接近1 = 越异常
```

**为什么选余弦距离而不是欧氏距离？**

余弦距离只看**方向**，不看**长度**。两张光照不同的同一产品照片，它们的特征向量的"长度"可能差很多（亮度不同导致激活值差异），但"方向"接近（纹理模式相似）。如果用欧氏距离，亮度变化直接拉大距离——假阳性暴增。

### 1.5 50类基线结果：哪些品类"天生复杂"？

全量50类推理耗时 **6分42秒**（RTX 4090）。所有分数都在 0.3 以下——正常样本之间的一致性比想象中高。

**整体统计：** 分数范围 0.044 ~ 0.202，均值 0.133

**🔝 异常分数最高的5个类别（天生差异大）：**

## ❓ 新问题

DINOv2 给了你一个数字——0.133。它告诉你"整体看起来还行"。但比赛里 50% 的分数在**分割**上——你必须画出一个热力图，精确到像素级别："左下角 0.87 异常，右上角 0.02 正常"。

DINOv2 的 768 维全局向量做不到这一点。但 DINOv2 内部其实有更丰富的信息——它在 12 层 Transformer 的每一层都维护着 196 个位置的局部特征。下一章，打开这个黑箱。

**→ Ch2: 多尺度异常定位 — 从全局分数到像素级热力图**
